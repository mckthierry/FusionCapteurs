## 4. ANALYSE D'ALIGNEMENT NEUROSYNAPSE ↔ DÉFI B (Fusion de capteurs fiable)

Même si NEUROSYNAPSE a été rédigé pour le Défi A, il est pertinent d'analyser son
alignement avec le Défi B pour identifier les gaps à combler si l'équipe veut postuler.

### 4.1 Points de convergence forts

Fusion multi-capteurs ISR :
  NEUROSYNAPSE : radar RF, EO/IR, SIGINT, texte tactique (4 modalités)
  Défi B : radar, EO/IR, SIGINT, UAS, acoustique distribué (convergence forte)
  Verdict : ALIGNÉ — la proposition couvre les capteurs prioritaires du Défi B

Contraintes embarquées (SWaP) :
  NEUROSYNAPSE : < 220 Ko mémoire, ≤ 50 ms latence, Jetson Orin NX
  Défi B : systèmes militaires mobiles sans connectivité centralisée
  Verdict : ALIGNÉ — les contraintes SWaP sont explicitement adressées

Fiabilité et incertitude :
  NEUROSYNAPSE : MC Dropout N=30, ECE < 0.05, indicateur HAUTE/MOYENNE/FAIBLE
  Défi B : "IA fiable" est le terme central du défi
  Verdict : ALIGNÉ — l'estimation d'incertitude épistémique répond à "fiable"

Traçabilité et auditabilité :
  NEUROSYNAPSE : SHA-256 par décision, E5 audit, Directive GC automatisée
  Défi B : "compromised sources, methods, operations" si erreur classification
  Verdict : ALIGNÉ PARTIEL — SHA-256 est présent mais classification
  multiniveaux (PROTÉGÉ B / NON CLASSIFIÉ) déclarée hors périmètre Phase 1

### 4.2 Gaps identifiés par rapport au Défi B

GAP-FC-01 — Classification multi-domaines sécurisée :
  Défi B mentionne "combining data from multiple classification domains"
  avec risque de compromission de sources. NEUROSYNAPSE déclare ce point
  explicitement HORS PÉRIMÈTRE Phase 1 (DEC-S7-01, CCP-6 corr. S7).
  Sévérité : HAUTE si Défi B l'exige en Phase 1.

GAP-FC-02 — UAS et capteurs acoustiques distribués :
  Défi B cite les UAS et capteurs acoustiques distribués comme sources.
  NEUROSYNAPSE couvre radar RF, EO/IR, SIGINT, texte mais PAS l'acoustique
  ni les flux UAS autonomes. Audio tactique/sonar déclarés Phase 2 (DEC-S7-02).
  Sévérité : MOYENNE — le mécanisme de substitution E1 permet extension.

GAP-FC-03 — Budget Défi B vs NEUROSYNAPSE :
  Source search CanadaBuys : "build phase TRL 6-9, up to $5 million"
  NEUROSYNAPSE budget : 250 000 CAD (Défi A, plafond différent)
  Si Défi B a un plafond différent, le budget de la proposition doit être
  recalibré. La structure de coûts (3 ressources, taux horaires) reste valable.
  Sévérité : MOYENNE — recalibrage requis, pas de refondue architecturale.

GAP-FC-04 — NMT/TRL d'entrée et de livraison :
  NEUROSYNAPSE : NMT-4 → NMT-5 en 6 mois (Défi A)
  Défi B (search) : TRL 6-9 mentionné pour la phase de construction
  Si Défi B exige un TRL d'entrée plus élevé (ex. TRL 6 vs NMT-4),
  NEUROSYNAPSE devrait démontrer une maturité plus avancée à l'entrée.
  Sévérité : HAUTE — potentiellement éliminatoire si les niveaux ne correspondent pas.

GAP-FC-05 — Données réelles vs synthétiques (Limite L1) :
  NEUROSYNAPSE déclare explicitement Limite L1 : validation sur données
  synthétiques haute-fidélité uniquement (NMT-5 sur synthétique).
  Défi B : "pour des missions réelles" est dans le titre même du défi.
  La Limite L1 pourrait être rédhibitoire si Défi B exige une validation
  sur données opérationnelles réelles en Phase 1.
  Sévérité : CRITIQUE — directement contradictoire avec l'intitulé du défi.

## 6. PERSONA ADVERSARIAL RAPIDE — 4 ANGLES

### Angle A — Ingénieur systèmes sceptique
"Vous prétendez couvrir 'missions réelles' mais toute votre validation est
synthétique. Comment justifiez-vous NMT-5 sans une seule mesure sur données
capteurs réels ? Le Défi s'appelle littéralement 'pour des missions réelles'."

### Angle B — Évaluateur MDN
"La classification multi-domaines est le cœur du problème de fusion sécurisée
au MDN. Votre défi B mentionne explicitement le risque de compromission de sources.
Déclarer cela 'hors périmètre' n'est pas une réponse acceptable si l'évaluateur
note CCP-6 sur cet alignement précis."

### Angle C — Chercheur ML pair
"La différence entre Défi A et Défi B n'est pas juste de terminologie.
'Fiable' dans le titre implique des métriques de fiabilité terrain, pas
uniquement ECE < 0.05 sur données synthétiques. Où est la validation adversariale
sur scénarios opérationnels dégradés (brouillage, occultation, latence réseau) ?"

### Angle D — Opérateur ISR
"Je dois décider si oui ou non j'engage. Votre système me donne JAUNE = ALERTER
sur un scénario où deux capteurs sont actifs sur données synthétiques.
Sur le terrain, avec UAS dégradé et SIGINT bruité, que se passe-t-il ?
Je n'ai aucune réponse à ce risque dans votre proposition actuelle."
